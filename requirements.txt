#   2) Install PyTorch / torchvision / torchaudio from the cu121 wheel index

#
# Why PyTorch is not listed here:
#   The official cu121 wheels live on download.pytorch.org, not PyPI. Installing
#   torch first guarantees a CUDA build; a plain `pip install torch` on Windows
#   often pulls a CPU-only wheel and breaks AutoGPTQ runtime.
#
# Triton (avoid on Windows):
#   AutoGPTQ exposes Triton only as an *optional* extra (`auto-gptq[triton]`).
#   Do not install that extra on Windows — Triton is aimed at Linux and tends
#   to fail or conflict. The default `auto-gptq` install does not require Triton.
#   Older AutoAWQ releases (e.g. 0.2.1–0.2.4) listed `triton`; if you opt into
#   AutoAWQ manually, use a release with `autoawq-kernels` win_amd64 wheels.
#
# Transformers / tokenizers:
#   transformers==4.37.2 requires tokenizers >=0.14,<0.19. We constrain
#   tokenizers explicitly so a dependency bump cannot break that invariant.
#
# AutoAWQ on Windows (optional):
#   AWQ is intentionally optional in this project due to import compatibility
#   constraints on some Windows + transformers combinations. If you enable AWQ,
#   prefer autoawq==0.2.5 (0.2.6 pins torch==2.3.1).
#
# NumPy:
#   Many torch 2.2.x + quantization stacks were validated on NumPy 1.x. We cap
#   NumPy <2 to reduce binary ABI surprises on Windows.
